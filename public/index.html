About CRMS

CRMS (Class Representative Management System) is a small web application that digitalizes classroom attendance and reduces proxy marking for lab sessions.

The system is designed for a typical university lab environment where the class representative is responsible for collecting attendance. Instead of passing paper sheets, each student logs in with a unique roll number and password.

Every session is opened and closed by the CR, and the system records which device was used for each student. Suspicious entries can be flagged, reviewed, and corrected from the admin dashboard.

Key Features

The project focuses on combining a clean dark UI with simple but meaningful back‑end checks.

Secure login

Students register with roll number and password. Credentials are stored with hashing in the MySQL database.

Session-based rules

Each attendance session has a unique code. A student can mark attendance only once per session from a single device.

Admin dashboard

The CR can open and close sessions, view class stats, and inspect flagged attendance records from a separate admin interface.

How it works

  1. CR logs into the admin portal and opens a new attendance session.
  2. Students log into the student portal and mark their presence for that session.
  3. The system enforces per‑session and per‑device checks and highlights suspicious entries.
